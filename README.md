# 🌟 `CLI foundation` — Advancing the Future of Command‑Line Interfaces

Welcome to **cli-foundation** — an open, community‑driven initiative dedicated to improving the design, usability, consistency, and overall experience of command‑line interfaces (CLIs).

CLI tools are powerful, flexible, and loved by developers — but across ecosystems, they vary wildly in structure, terminology, help text, error formats, and output style. This diversity can be useful, but it also creates friction:

*   Users must continually learn new conventions
*   Important UX lessons are rediscovered repeatedly
*   Tooling ecosystems feel fragmented
*   Accessibility and readability vary significantly

The **cli-foundation** exists to help change that — collaboratively, inclusively, and transparently.

***

## 🌱 Mission

Our mission is to:

### **“Improve the usability, clarity, and consistency of CLI tools across all ecosystems.”**

We aim to bring together developers, tool maintainers, designers, writers, and anyone passionate about command‑line interfaces to:

*   Share best practices
*   Develop design guidelines
*   Offer examples of clear, consistent CLI output
*   Explore UX principles unique to text‑based interfaces
*   Create optional, community‑maintained standards
*   Encourage convergence around patterns that benefit everyone

This foundation is **not** an authority — it is a **home** for collaboration and learning. Authority, if ever earned, will emerge from adoption, not enforcement.

***

## 🌍 Vision

We envision a world where:

*   CLI tools feel familiar, even when built by different teams or languages
*   Help text and error messages follow recognizable patterns
*   Output formats are predictable and well structured
*   Tools are accessible, readable, and friendly by default
*   Developers have a trusted reference for building excellent CLI experiences

And ultimately:

> **A shared, community‑adopted set of CLI standards emerges naturally from open discussion and contribution.**

***

## 🧱 What You Will Find Here

The `cli-foundation` organisation hosts several repositories with different purposes, including:

### 📘 **cli-standards**

A long‑term effort to define an open, optional, community-built standard for CLI output, structure, and behavior.

### 📙 **cli-guidelines**

Practical best practices, design notes, and explanations for building better CLIs.

### 📗 **cli-examples**

Real‑world examples of good and bad CLI design, with explanations and comparisons.

### 🎨 **cli-style-guide**

Human‑friendly recommendations for formatting, color usage, tone, documentation style, and accessibility.

### 🧪 **cli-testing-tools**

Experimental tools and methods for evaluating CLI output consistency, human readability, and UX behavior.

These repositories will evolve organically with community involvement.

## 🧭 Guiding Principles

To keep this initiative grounded and collaborative, we follow these principles:

### **1. Community over authority**

Nothing here is imposed. Everything is suggested, discussed, and improved through consensus.

### **2. Practical over theoretical**

Real-world examples and clear explanations take priority over rigid theory.

### **3. Clarity over cleverness**

Simple, readable, user-friendly design wins.

### **4. Accessibility for all users**

CLI tools should be as inclusive and readable as possible.

### **5. Evolution through adoption**

Standards will emerge only if the community finds them genuinely helpful.
